Bank Audit Platform

Internal audit automation web app that turns monthly compliance checks into downloadable main files (Excel/Word/ZIP), with traceable rules, repeatable runs, and reduced manual workload across audit teams.

What it automates (modules)

  • Cuentas Inmovilizadas → TGN (Cruce): Upload CA/CC captaciones + operaciones TGN + “Mayores 500” + fechas → generates a consolidated Excel (sheet CRUCE) and supporting analysis sheets for audit review.
  • Retenciones: Runs the retentions flow and produces standardized artifacts: Retenciones_YYYYMMDD.xlsx, Extractos_CtasConRetencion_YYYYMMDD.xlsx, and PorCuenta_YYYYMMDD.zip.
  • Fallas de Caja: Calculates metrics/saldos for the period and generates audit workpapers: A2 Análisis Contable (Excel) and A4 Verificación de Comprobantes (Excel) from voucher evidence.
  • Anexo 1 — Activos de Primera Calidad: Upload Situación Patrimonial + Cartera Calificación + ASF-510 + Central O/P → fills the template and outputs Anexo_1_DDMMYYYY.xlsx.
  • Encaje Legal: Upload captaciones (CA/CC/DPF) + Situación Patrimonial + Encaje Legal → builds Encaje_Legal_AAAA_MM.xlsx and includes rule-based calculations/insertions used in the audit paper.

Process (how a run works)

  • Input validation: controlled file types, required fields, date parsing, and predictable naming.
  • ETL execution: fast extraction (incl. Polars where it matters), transformation/merges (Pandas), and database pulls (Oracle/SQLite) for reconciliation.
  • Controls & traceability: consistent logging, reproducible outputs, and purge/clean of output folders per run to avoid mixing periods.
  • Template-driven deliverables: writes into pre-approved Excel templates (OpenPyXL/XlsxWriter) and generates Word reports via Windows Word COM when required.
  • Evidence handling: supports voucher packs in .docx and converts them into structured verification tables for audit documentation.

Smart extraction (AI-assisted evidence)

  • Voucher parsing from images: extracts key fields (date, receipt number, amounts, glosa) using a local vision model (Ollama) with strict JSON outputs (LangChain parsers).
  • Glosa normalization: uses a LangGraph flow to detect and standardize dates inside free-text “glosa” (reduces manual interpretation errors).
  • Result: faster A4 verification and fewer “copy/paste” mistakes when building the paper trail.
